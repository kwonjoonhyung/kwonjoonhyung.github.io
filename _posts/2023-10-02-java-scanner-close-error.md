> 발생 개요

자바 프로그램을 작성하다가 다음과 같은 오류가 어느 날 떴다.

![scannerError](/public/img/NoSuchElementException.png)

특정 메서드를 돌고 난 이후에 메인 함수에서 스캐너에 문제가 생긴 거 같은데 해당 오류 발생 직전 메서드 내에 스캐너객체명.close();를 삭제해 주니 오류가 없이 정상적으로 작동했다.

> 발생 원인

Scanner를 닫아서 문제가 생기는 것이 아닌 정확히는 System.in이 닫혀서 생기는 오류이다.

이유는 Scanner 객체를 여러 개 만들어도 입력을 받아주는 System.in 객체는 여러 스캐너가 전부 공유하고 있다.

그런데 앞에서 Scanner를 닫아버리면 System.in 객체도 함께 닫히기 때문에 이후에 사용되는 Scanner에서 문제가 발생하는 것이다.

> 해결 방법

여러 가지 방법이 존재하는데 스택오버플로우 기준 가장 따봉이 많이 박힌 답변은 그냥.. '입력 스트림을 사용해야 하는 코드에서는 스캐너를 웬만하면 닫지 마라'였다.

즉, 스캐너객체명.close(); 를 삭제하여 해결할 수 있다. 물론 메인 함수가 끝나기 직전에는 더 이상 입력 스트림을 사용할 일이 없으니 닫아줘야 한다.

두 번째는 scanner 객체를 닫을 때 입력 스트림(System.in)은 닫히지 않게 막아주는 코드가 있다.

ex) CloseShieldInputStream

근데 여기서 들 수 있는 의문은 Scanner를 닫지 않으면 불필요한 스캐너들 정리가 안되지 않나? 그래서 최적의 방법은 그냥 스캐너를 하나 만들고 공유하여 돌려쓰는 방법이 좋은 거 같다.
