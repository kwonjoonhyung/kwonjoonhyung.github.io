# 무결성 제약 조건의 종류

도메인 무결성 제약 조건

엔티티 무결성 제약 조건

참조 무결성 제약 조건

> 도메인 무결성 제약 조건

각 애트리뷰트의 값은 반드시 원자 값이어야 한다.

투플들이 각 속성의 도메인에 지정된 값 만을 가져야 한다.

SQL 문으로 데이터 타입을 정하고 널, default 값, 체크 등을 사용하여 추가로 제약 조건을 걸 수 있다.

> 엔티티 무결성 제약 조건

릴레이션의 기본 키를 구성하는 애트리뷰트는 널 값을 가질 수 없다.

> 참조 무결성 제약 조건

아래의 두 가지 조건 중 하나를 만족해야 한다.

외래 키의 값은 참조되는 릴레이션 내에 특정 튜플의 기본 키 값과 같다.

외래 키가 본인 릴레이션 내에서 기본 키의 구성요소가 아니라면 널 값을 가진다.

즉, 정리하면 참조되는 릴레이션의 특정 튜플의 기본 키를 참조하되, 존재하지 않는 값을 참조할 수는 없다. 존재하지 않는 경우는 널 값을 넣어주면 되는데 이 또한 물론 기본 키의 구성요소가 아니라는 가정하에 가능하다.

> 주의 사항

삽입 연산 시 위 세 가지 제약 조건을 모두 위배할 수 있으므로 주의한다.

삭제 연산 시 참조되는 릴레이션인 경우 참조 무결성 제약 조건을 위배할 수 있다.

> 참조 무결성 제약 조건을 지키기 위한 DBMS 옵션

restricted : 제약 조건을 위배하는 연산을 거절

cascade : 참조되는 릴레이션에서 삭제 시 제약 조건에 위배되면 참조하는 튜플도 함께 삭제

nullify : 위와 같은 상황에서 참조하는 튜플을 삭제하는 대신 외래 키에 널 값을 대입

default : 위와 같은 상황엥서 널 값 대신 default 설정값을 넣음
